# 全自动写代码，博客全自动改造，一句话生成表情包动态图标，CodeX太好用了，我已经不自己写代码了

这个视频描述一下我最近都用CodeX干了什么，说实话是非常惊艳的，感觉就是太爽了。

那种感觉就像什么呢？不亚于最开始学习编程，最开始用gpt，刚学会翻墙的那种打开新世界的感觉。

我干了什么呢？这个视频列举我这两三天用CodeX做的事情。主要就是三个小例子和我使用AI后的一些感受。

首先第一个给大家看个小例子，一句话生成动态图标生成表情包：

AI生图的能力，skill阅读能力可见一斑。之前要是要制作表情包的话或者这种logo以及加动态效果怎么着也得几百块钱，现在只需要打打字，虽然说有些细节AI还是不能很好处理，比如说生成表情包的时候，我说不要有手，手用完美的圆球就行但是AI根本理解不了我说的是什么意思。

第二个例子就是工作上，像那种比较简单的数据采集任务，直接一句话给他描述，他就嘎嘎干了，他自己会去看页面，写代码请求提取，然后运行测试，我连运行都不用，我再打开检查一下数据就可以了。之前我有出过一个自动化数据采集的合集，说可以搞定80%的数据采集需求，现在AI自己已经可以搞定80%的需求了。然后也有一些Codex搞不定的，就是很少，比如说流程比较复杂点的，其实我感觉他还是能实现的，只不过可能因为我没给他描述清楚，或者说浏览器没有提供对应的接口工具给他调用没有让他不能发挥出完全的能力。这时候把需求给ai描述清楚然后再等AI试错的时间可能等太久所以不如自己写了。

这个给大家说起来还是比较抽象的，因为大家可能用不到，大家可能感受不到我当时使用的时候的震撼，但是要是你真的用CodeX，CloudCode写过代码的话，肯定是理解我的。

第三个例子就是改造博客，在最开始搭建这个博客的时候就是纯手搓出来的，后面用gpt对话的那种改造了下样式，那时候ai写代码还是很低效效果也是很局限的。但现在用CodeX已经完全不一样了，首先就是他能够全自动自主提建议然后修改还提交代码。手动写代码到网页对话到现在agent，三个时代就是农业时代到工业时代再到互联网时代的感觉。

我当时的博客我是自己部署了一套服务，自己写的代码，可以对博客增删改查，还卖vip，然后用到的图片会自动上传到我的minio-oss服务里面，后面网站没精力维护了之后我就想把文章迁移到我的博客来着。迁移差不多的流程就是：得找到我系统中有但是博客中没有的文章，复制粘贴出来，检查文章的样式，把用的oss的图片复原出来，然后文章使用的图片链接改成本地的。图片、文章的英文名字取什么。工作量是很大的，得一直复制粘贴，就是很低级的重复性工作。可能也可以写代码自动迁移，但是准确性又难以保障。

像minio-oss存储格式那些还需要我自己去研究，手动做虽然说能做，但是肯定要消耗很多精力和时间，投入回报不成正比，所以我就没做。只是把数据被备份下来了。我当时想的就是等以后AI强大了让AI帮我做，但当时还是网页对话框聊天的形式，还是很难以想象的，没想到过了一年，现在agent真的可以实现了，我就在躺床上刷着抖音，等着他干活就行了。

最后再说说我现在对于ai编程的看法。

虽然说啊，AI极大的提升了工作的效率，但是我感觉其实工作真的是更累了。因为本来可能要三五个人才能做的事情，现在一个人就可以做了。关键不是做的事情多了所以累了，而是本来三五个人背的锅，现在全是你一个人背锅了。如果遇到那种傻福领导，不理解打工人，特别自私的话，想着有了AI打工人就能干的很快，一直催着干的话，把所有的活都甩给你，确实挺难受的。

就像有一些任务，明明领导就可以一句话命令AI去做，但是领导不会这样做。领导会让你去做，然后你去用ai做然后汇报给领导。出了问题领导会继续找你，领导肯定是不想负责的，AI也不负了责，所以打工人就成为了一个专业背锅中间件。

上面就是我对AI目前发展的用处和理解。大家有不同的想法，可以在评论区讨论。
