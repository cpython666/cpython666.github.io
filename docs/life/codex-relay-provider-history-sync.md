# Codex 如何使用中转站？历史记录不显示如何解决？好用的中转站推荐

最近使用 Codex 编程很爽啊，自己根本就不手动写代码了。

但是发现很容易被 5 小时额度卡脖子，五个小时的额度，一个小时就用完了，剩下四个小时干瞪眼。因为 AI 的水平毕竟是比较高的，我这种小菜货想看懂还是有点难度的，想看懂代码还是得让 AI 跟我解释一遍，但是问题是 AI 没额度了啊。

所以只能曲线救国了，而且我们经常能看到中转站说自己的比较便宜，也有很多人用，所以打不过就加入，为什么不试一试呢？

所以我今天就试了试用这个中转站，不试不知道，试了才知道小问题还是比较多的，还是比较考验操作的。

所以今天给大家一个沉浸式体验，一步一步演示如何切换到中转站。

Windows 和 macOS 流程操作都是一样的。

首先我们总结一下将要进行的流程：

- 下载 CC Switch。因为 Codex 默认走官方 API，它自己的软件又不支持直接指定使用别的 API，所以需要使用第三方软件，在里面配置 API 信息，包括中转站的请求链接和你的密钥。这里我发现了一个比较好用的中转站，我将邀请链接放到视频简介和评论区了，大家可以用一下，小小支持一下 UP。
- 配置完成并重启 Codex 后，会发现对话历史记录看不到了。因为 Codex 是用接口提供商的名字分组聊天记录的，所以换了提供商，聊天记录就看不到了。我们依旧需要一个第三方软件，它会识别本地聊天记录并进行迁移，一会儿会演示这个操作。
- 主要就是上面两步。目前我的电脑还是纯官方版本，接下来就演示如何换成中转站。
