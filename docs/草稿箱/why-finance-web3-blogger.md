# 为什么我转行做财经类和 Web3 博主？

最近我一直在想一个问题：

我为什么要转去做财经类和 Web3 方向的内容？

说实话，不是因为我突然悟道了，也不是因为我已经是什么金融大佬了。

我承认，我现在依旧是个大菜货。

只是我后来发现一件很现实的事情：

**普通人想要破局，不能一直待在最卷、最穷、最没有溢价的地方。**

你要去风口。

你要去不缺钱的地方。

比如游戏产业，比如金融，比如 Web3。

这些地方有一个很明显的特点：

**里面的人是真的有钱，也是真的愿意为认知、工具、信息、效率付费。**

说白了，你在一个大家都没钱的地方，再努力也很难赚到钱。

就像你在学校门口卖几千块的高端咨询，大家可能觉得你疯了。

但你在金融市场里，别人亏一次就是几万、几十万，他花几百块买一个工具，买一份信息，买一个认知，反而会觉得很正常。

这就是场景不一样。

以前我做编程内容，其实也挺开心的。

写教程，做项目，分享代码，看到有人因为我的内容学会东西，我也会很有成就感。

但是后来我慢慢发现，Web2 的技术内容已经非常卷了。

Python 教程有人讲。

前端教程有人讲。

爬虫教程有人讲。

AI 教程现在更是一堆人讲。

你今天刚想做一个选题，明天可能就有十个人已经发了。

而且更可怕的是，AI 对这些内容的冲击也很大。

以前你不会写代码，要去搜教程，要去看视频。

现在你直接问 AI，它能给你代码，能给你解释，甚至能帮你改 bug。

当然 AI 不能完全替代学习，但它确实会让大量基础教程的价值变低。

这就是现实。

不是我不喜欢写代码了，而是我发现，单纯写 Web2 技术教程这条路，越来越像在红海里面游泳。

大家都在里面扑腾。

水花很大。

但鱼可能没多少。

而 Web3 不一样。

Web3 本质上是开放的协议。

链上数据是公开的，资产是流动的，钱包是开放的，协议之间可以互相组合。

它不像 Web2，很多东西都被平台锁住。

你想接某个平台的数据，别人接口一关，你就没了。

你想做某个平台的生态，别人规则一变，你就得重来。

你想在平台上做内容，账号可能说没就没。

但 Web3 里面，很多东西是公开摆在那里的。

链上发生了什么，谁买了什么，资金流向哪里，协议怎么运转，很多东西你都可以去看，去分析，去做工具，去做内容。

这对程序员来说，其实是很有吸引力的。

因为程序员最怕什么？

最怕自己有技术，但是没有场景。

你会写代码，但是不知道给谁用。

你会做工具，但是不知道谁愿意付钱。

你会分析数据，但是不知道分析什么有价值。

而 Web3 刚好给了很多场景。

交易数据、链上数据、钱包监控、空投工具、项目分析、风险识别、自动化、资讯聚合、策略回测。

这些东西都需要技术，也需要认知。

这不就是程序员可以切进去的地方吗？

当然，我说 Web3 是蓝海，不代表进去就能发财。

更不代表你一进去就梭哈合约，然后三天财富自由。

那大概率不是财富自由，是钱包自由落体。

我自己也被市场教育过。

知道和做到是两码事。

看懂机会和抓住机会也是两码事。

Web3 里面机会多，风险也大。

它不会因为你是新人就对你温柔一点。

甚至你越是新人，越容易被各种暴富故事、内幕消息、百倍神话吸进去。

所以我现在转财经和 Web3，不是为了装自己很懂。

而是我觉得这个方向值得长期学习，长期记录，长期输出。

我做内容，本质上也是在记录自己的认知变化。

我现在看不懂的东西，过半年可能能看懂一点。

我现在踩过的坑，后面也许能提醒别人少踩一点。

我现在写下来的这些东西，未来回头看，可能会觉得自己当时很幼稚。

但没关系。

至少我在路上。

我越来越觉得，普通人做事不能只看自己会什么，还要看这个世界哪里有需求。

编程是我的技能。

财经和 Web3 是我想进入的场景。

技术不能一直待在技术里面自嗨。

技术要去有钱的地方，去有需求的地方，去变化最快的地方。

所以我转行做财经类和 Web3 博主，不是因为我已经成功了。

恰恰相反，是因为我还没有成功。

我想去更大的水域里看看。

我想看看一个普通程序员，能不能靠持续学习、持续记录、持续表达，在新的行业里找到自己的位置。

这就是我当前阶段的理解。

不一定对。

但我觉得值得试。

因为想的时候全是问题，做的时候才有答案。
